# 🪒 Pirayesh — ربات تلگرام آرایشگاه مردانه

ربات هوشمند نوبت‌گیری و مدیریت آرایشگاه با پشتیبانی از پرداخت کارت‌به‌کارت و پنل مدیریت کامل.

---

## ✨ امکانات

### کاربر
- 📅 نوبت‌گیری آنلاین (انتخاب خدمت → تاریخ → ساعت → تایید)
- 💳 پرداخت کارت‌به‌کارت با ارسال رسید
- 💵 پرداخت حضوری
- 📋 مشاهده و لغو نوبت‌های فعال
- 📞 مشاهده اطلاعات آرایشگاه

### ادمین
- 📋 مشاهده نوبت‌های امروز و همه نوبت‌ها
- ✅ تایید / رد / لغو / انجام‌شدن نوبت
- 💳 تایید یا رد رسیدهای پرداخت
- ✂️ مدیریت خدمات (افزودن، فعال/غیرفعال)
- 🕐 مدیریت زمان‌بندی (تعریف روز و ساعت‌های کاری)
- 🏪 ویرایش اطلاعات آرایشگاه از داخل ربات
- 🔧 toggle فعال/غیرفعال کردن بخش‌ها
- 📊 آمار کلی نوبت‌ها

---

## ⚙️ تنظیمات قابل toggle از پنل مدیریت

| بخش | توضیح |
|-----|-------|
| 🟢/🔴 نوبت‌گیری | فعال یا غیرفعال کردن دکمه نوبت‌گیری برای کاربران |
| 🟢/🔴 پرداخت آنلاین | نمایش یا مخفی کردن گزینه ارسال رسید |
| 🟢/🔴 نمایش خدمات | فعال یا غیرفعال کردن منوی خدمات و قیمت‌ها |

---

## 🏪 اطلاعات آرایشگاه (از پنل مدیریت)

همه موارد زیر را می‌توان از داخل ربات (⚙️ پنل مدیریت ← 🏪 اطلاعات آرایشگاه) تغییر داد:

- نام آرایشگاه
- شماره تلفن
- آدرس
- شماره کارت بانکی
- نام صاحب کارت

## 🚀 نصب روی VPS

### پیش‌نیاز
- Ubuntu 20.04 / 22.04
- دسترسی root یا sudo
- توکن ربات از [@BotFather](https://t.me/BotFather)
- آیدی عددی ادمین (از [@userinfobot](https://t.me/userinfobot))

### نصب یک‌مرحله‌ای

```bash
git clone https://github.com/moha100h/Pirayesh.git
cd Pirayesh
chmod +x install.sh
./install.sh
```

اسکریپت فقط دو چیز می‌پرسد:
1. **توکن ربات** — از @BotFather بگیرید
2. **آیدی عددی ادمین** — از @userinfobot بگیرید

بقیه تنظیمات (نام آرایشگاه، تلفن، آدرس، شماره کارت) را از **پنل مدیریت ربات** انجام دهید.

## 🐳 مدیریت Docker

```bash
# مشاهده لاگ
docker compose logs -f

# ری‌استارت
docker compose restart

# توقف
docker compose down

# آپدیت بعد از تغییر کد
git pull && docker compose up -d --build
```

## 🗄️ دیتابیس

SQLite (پیش‌فرض) — فایل `pirayesh.db` در همان پوشه پروژه ذخیره می‌شود.  
برای محیط production می‌توانید `DATABASE_URL` را در `.env` به PostgreSQL تغییر دهید:

```
DATABASE_URL=postgresql+asyncpg://user:pass@localhost/pirayesh
```

## 🔄 فلو نوبت‌گیری

```
کاربر: 📅 نوبت‌گیری
  └─► انتخاب خدمت
        └─► انتخاب تاریخ
              └─► انتخاب ساعت
                    └─► تایید نوبت
                          └─► پرداخت (کارت‌به‌کارت یا حضوری)
                                └─► ارسال رسید ──► ادمین تایید/رد می‌کند
                                                        └─► اطلاع‌رسانی به کاربر
```

---

## 🔄 فلو پرداخت

```
کاربر رسید ارسال می‌کند
  └─► ادمین نوتیف دریافت می‌کند
        └─► ✅ تایید → نوبت قطعی + اطلاع به کاربر
        └─► ❌ رد    → اطلاع به کاربر برای ارسال مجدد
```
